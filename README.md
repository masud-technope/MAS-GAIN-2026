# Powerful LLMs are not Enough: Understanding Interaction Faults in Multi-Agent Systems

**Mohammad Masudur Rahman** and **Asif M. Samir**

**Abstract**: Given the large expenditures on AI infrastructure, the construction of bigger and more powerful foundation models has been a top priority for the AI industry. However, in this position paper, we argue that powerful LLMs might not be enough to develop reliable, dependable multi-agent systems. An in-depth understanding of their system-level challenges, especially agent interaction failures, is warranted. We construct a multi-agent system with five specialized agents for code review, inject three types of agent interaction faults, conduct experiments, and present strong evidence supporting our position. We also discuss counter-evidence and finally conclude the paper with a call to three key actions. 

# Included Materials
The repository contains data, code, and notebooks for collecting, processing, generating, and evaluating code reviews.

# Contents
- `collect_pr_data.py` — script used to collect PR/review data (entry point for data collection).
- **Notebooks**:
  - `CodeReviewCollector.ipynb` — captures PR/review metadata from the raw JSON files.
  - `CodeReviewMAS.ipynb` - baseline version of the multi-agent system performing code reviews.   
  - `CodeReviewMAS_Faulty.ipynb` — faulty version of the multi-agent system performing code reviews. It contains three types of agent interaction faults: task delegation fault, message lost fault, and message corruption fault.
  - `ReviewEvaluator.ipynb` - evaluates the quality of code reviews produced by the multi-agent system. 
  - `ResultAnalyzer.ipynb` — analyzes the results of code review evaluation.
- `dataset/` — raw and processed data
  - `raw/` — original JSON PR files organized by project (`fastapi`, `pandas`).
  - `processed/` — normalized, sanitized JSON outputs (e.g. `fastapi.json`, `pandas.json`).
  - `faulty/` — code reviews generated by the faulty variants of the multi-agent system. LLM core: `GPT-4o-Mini`
  - `reviewed/`, `reviewed-ext/` — code reviews produced by two baseline versions of the multi-agent system. LLM cores: `GPT 3.5-Turbo` and `GPT-4o-Mini`
  - `result-compare/`, `result-compare-ext/` — comparison results between two baseline versions and the faulty versions of the multi-agent system.
  - `trace/` — JSONL traces of agent interactions from five versions of the multi-agent system.
